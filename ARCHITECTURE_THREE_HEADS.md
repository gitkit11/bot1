# Архитектура Chimera AI: "Три Головы"

## 📋 Суть идеи

**Chimera AI** — это не просто бот, а **совет трех независимых ИИ-разумов**, каждый из которых имеет свой уникальный "голос" и способ мышления.

Точность прогноза достигается благодаря **конфликту мнений** — когда разные модели дают разные оценки, это помогает выявить неопределенность и риски.

---

## 🧠 Три Головы (Агенты)

### 1️⃣ **СТАТИСТИК (GPT-4o)**
**Роль:** Анализирует ТОЛЬКО цифры и вероятности.

**Входные данные:**
- Вероятности из нейросети "Пророк" (П1, Х, П2)
- Форма команд (последние 5 матчей)
- Статистика голов (xG, среднее голов)

**Выходные данные:**
```json
{
  "analysis_summary": "Хозяева имеют явное преимущество по статистике...",
  "home_win_prob": 0.68,
  "draw_prob": 0.14,
  "away_win_prob": 0.18,
  "match_balance": "явный фаворит хозяева"
}
```

**Логика:** Чистая математика. Если цифры говорят П1 с 68% — Статистик это подтвердит.

---

### 2️⃣ **РАЗВЕДЧИК (GPT-4o)**
**Роль:** Анализирует качественные факторы, невидимые в статистике.

**Входные данные:**
- Новости о командах (травмы, конфликты, мотивация)
- Настроение в прессе
- Информация о ключевых игроках

**Выходные данные:**
```json
{
  "analysis_summary": "Новостной фон благоприятен для хозяев...",
  "home_team_sentiment": 0.6,
  "away_team_sentiment": -0.2,
  "key_factor": "Ключевой игрок гостей восстановился после травмы"
}
```

**Логика:** Если новости говорят, что П1 слабее, чем показывает статистика — Разведчик это выявит.

---

### 3️⃣ **АРБИТР (GPT-4o)**
**Роль:** Синтезирует мнения Статистика и Разведчика, выносит финальный вердикт.

**Входные данные:**
- Отчет Статистика
- Отчет Разведчика
- Коэффициенты букмекеров

**Выходные данные:**
```json
{
  "final_verdict_summary": "Хозяева имеют явное преимущество...",
  "recommended_outcome": "Победа хозяев",
  "final_confidence_percent": 72,
  "expected_value_percent": 8.5,
  "recommended_stake_percent": 12,
  "bet_signal": "СТАВИТЬ",
  "signal_reason": "Высокая вероятность, хорошая ценность, низкие риски"
}
```

**Логика:** Взвешивает Статистику (60%) и Разведчика (40%), рассчитывает Value Bet и критерий Келли.

---

### 4️⃣ **LLAMA (Llama 3.3 70B через Groq)**
**Роль:** Независимое второе мнение, которое может не согласиться с тройкой выше.

**Входные данные:** Все то же самое, что и для Арбитра.

**Выходные данные:**
```json
{
  "analysis_summary": "Я вижу риск в этом матче...",
  "recommended_outcome": "Ничья",
  "home_win_prob": 0.55,
  "draw_prob": 0.28,
  "away_win_prob": 0.17,
  "final_confidence_percent": 45,
  "total_goals_prediction": "Больше 2.5",
  "both_teams_to_score_prediction": "Да"
}
```

**Логика:** Может дать совершенно другой прогноз. Это нормально! Конфликт мнений — это **фича**, не баг.

---

## 🔄 Как работает система

```
Матч: Everton vs Chelsea

    ↓
    
[СТАТИСТИК] → "П1 68%, это явный фаворит"
[РАЗВЕДЧИК] → "Но новости говорят, что Chelsea в хорошей форме"
[АРБИТР]    → "Синтезирую: П1 65%, ставка 12% по Келли"
[LLAMA]     → "Я не согласен, П1 только 55%, слишком рискованно"

    ↓
    
Вывод: Сигнал есть, но с низкой уверенностью (конфликт мнений)
```

## ⚠️ ВАЖНО: Нет Fallback на другую модель!

**ОШИБКА:** Если Llama недоступна (Groq 403), НЕ подменяем её на GPT!

**ПРАВИЛЬНО:** Возвращаем ошибку и сигнализируем об этом:
```
[Llama] ❌ Groq API вернула 403 (Access Denied).
Система работает в режиме "Трех голов" (без Llama).
Уверенность в прогнозе снижена.
```

**Почему?** Потому что Llama — это не "резервная копия GPT", это **третий независимый голос**. Если его нет — система должна это знать.

---

## 🎯 Когда система дает сигнал СТАВИТЬ?

1. **Уверенность >= 60%** (Арбитр согласен)
2. **Value Bet >= 5%** (есть реальная ценность)
3. **Критерий Келли > 0%** (рекомендуемая ставка больше нуля)
4. **Нет конфликта мнений** (или конфликт незначительный)

Если хотя бы один из этих критериев не выполнен → **ПРОПУСТИТЬ МАТЧ**.

---

## 🔧 Как исправить Groq 403?

Если Llama не работает:

1. **Проверьте API ключ:**
   ```bash
   echo $GROQ_API_KEY
   ```

2. **Проверьте региональные ограничения:** Groq может блокировать запросы из некоторых регионов.

3. **Проверьте лимит запросов:** Бесплатный план Groq имеет ограничения.

4. **Альтернатива:** Используйте другой провайдер для Llama:
   - OpenRouter (поддерживает Llama)
   - Together.ai
   - Replicate
