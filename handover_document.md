# Документация по передаче проекта Chimera AI

**Дата передачи:** 14 марта 2026 г.

**Цель проекта:** Продолжить разработку Telegram-бота Chimera AI для ставок на футбол и CS2.

## 1. Обзор проекта

Chimera AI — это гибридная система анализа ставок, использующая ансамбль математических моделей (ELO-рейтинги, распределение Пуассона, критерий Келли) и AI-агентов (GPT-4o, Llama 3.3 70B, Gemma 2). Система агрегирует данные из различных источников, включая The Odds API, API-Football и PandaScore (для CS2).

## 2. Реализованные изменения и улучшения

В рамках текущей фазы разработки были выполнены следующие ключевые задачи:

### 2.1. Улучшенная HLTV-интеграция и динамическое обновление статистики

*   **`hltv_stats.py`**: Файл `hltv_stats.py` теперь генерируется динамически с актуальными данными по винрейтам карт и рейтингам игроков, полученными с HLTV.org. Это обеспечивает более точную и своевременную информацию для анализа CS2-матчей.
*   **`scripts/update_hltv_stats.py`**: Разработан и обновлен скрипт `update_hltv_stats.py`, который использует Playwright для обхода Cloudflare и сбора данных с HLTV.org. Скрипт автоматически обновляет `hltv_stats.py` с последней статистикой команд и игроков.

### 2.2. Автоматизация MetaLearner

*   **`scripts/run_meta_learner.py`**: Создан скрипт `run_meta_learner.py` для автоматического запуска MetaLearner. Этот скрипт анализирует исторический ROI и предлагает корректировки порогов в `signal_engine.py`.
*   **Еженедельное расписание**: Настроено еженедельное выполнение скрипта `run_meta_learner.py` с помощью cron-задания. Это обеспечивает регулярное самообучение системы и адаптацию к изменяющимся условиям ставок.

### 2.3. Расширение сигналов CS2 в `signal_engine.py`

*   **Новые параметры в `CS2_CFG`**: В конфигурацию `CS2_CFG` добавлены новые параметры для более детального анализа CS2-матчей:
    *   `min_map_advantage`: Минимальное преимущество на картах (в процентах).
    *   `min_key_player_advantage`: Минимальное преимущество ключевых игроков (в процентах).
*   **Улучшенная логика `check_cs2_signal`**: Функция `check_cs2_signal` в `signal_engine.py` была расширена для учета новых параметров. Теперь она включает в себя оценку преимущества на картах и формы ключевых игроков, что делает сигналы CS2 более точными и надежными.
*   **Интеграция с `hltv_stats.py`**: `signal_engine.py` теперь использует данные из `hltv_stats.py` для расчета показателей, связанных с картами и игроками, что обеспечивает глубокую интеграцию и использование актуальной статистики.

## 3. Следующие шаги и рекомендации

Для дальнейшего развития проекта предлагаются следующие шаги:

1.  **Глубокая HLTV-интеграция**: Продолжить доработку `hltv_stats.py` для получения более детализированной информации о заменах игроков и их форме в реальном времени. Это может потребовать более сложного парсинга страниц HLTV или использования их API, если доступно.
2.  **Расширение видов спорта**: Добавить аналогичную логику и интеграцию данных для других дисциплин, таких как теннис, используя соответствующие источники данных и адаптируя модели анализа.
3.  **Мониторинг и оптимизация**: Регулярно отслеживать производительность MetaLearner и сигналов CS2, а также оптимизировать пороги и веса моделей для достижения максимального ROI.

## 4. Структура проекта и зависимости

### 4.1. Ключевые файлы

*   `meta_learner.py`: Модуль для самообучения и корректировки конфигураций.
*   `signal_engine.py`: Основной движок для генерации сигналов ставок.
*   `database.py`: Модуль для работы с базой данных `chimera_predictions.db`.
*   `agents.py`: Модуль, содержащий логику AI-агентов (GPT-4o, Llama 3.3 70B, Gemma 2).
*   `sports/cs2/hltv_stats.py`: Динамически генерируемый файл со статистикой HLTV.
*   `sports/cs2/veto_logic.py`: Логика симуляции вето карт и оценки их влияния.
*   `sports/cs2/core.py`: Ядро анализа CS2-матчей, объединяющее различные источники данных.
*   `scripts/update_hltv_stats.py`: Скрипт для обновления статистики HLTV.
*   `scripts/run_meta_learner.py`: Скрипт для запуска MetaLearner.
